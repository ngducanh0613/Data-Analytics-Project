Executive Summary

The Business Problem

While the Customer 360 platform stores OLTP data (contracts, registration, GMV), it lacked actionable analytical segmentation. Stakeholders needed answers to three core questions:

  • 1
    Which customers generate most of the revenue today?
  • 2
    Which groups are likely to grow in value if we nurture them?
  • 3
    Which customers are at risk of leaving if we do nothing?

The Solution

I leveraged Recency-Frequency-Monetary (RFM) analysis to score and map customers into four key segments based on an adapted BCG matrix concept.

VIP (Stars)

High Value / High Engagement

Thân Thiết (Cash Cows)

High Value / Solid Base

Tiềm Năng (Question)

High Growth Opportunity

Vãng Lai (Dogs)

Lowest Share / At Risk

Method of Analysis

Transforming raw OLTP data into structured business intelligence through a rigorous scoring logic.

1. Data Aggregation

Extracted data from Customer Register and Transactions. Filtered GMV = 0 (promotions) to reduce noise. Aggregated metrics up to Sept 2022.

2. IQR Scoring Logic

Calculated R, F, and M metrics per user. Used Interquartile Ranges to score each dimension from 1 (Lowest) to 4 (Highest).

3. Segment Mapping

Concatenated rankings (e.g., "424") and mapped them against a lookup table translating technical codes to 4 business-friendly BCG segments.

Appendix - SQL Code

Data engineering logic utilized to generate the OLAP output.

Dialect: T-SQL / Standard SQL
rfm_scoring_pipeline.sql 
-- STEP 1
-- BASE AGGREGATION
-- Calculate Contract Age, Recency, Frequency, Monetary per customer
WITH RFM AS (
  SELECT 
    cr.id,
    CASE 
      WHEN DATEDIFF(year, created_date, '2022-09-01') < 1 THEN 1.0
      ELSE DATEDIFF(year, created_date, '2022-09-01') 
    END AS Contract_age,
    DATEDIFF(day, MAX(ct.Purchase_Date), '2022-09-01') AS Recency,
    COUNT(ct.Transaction_ID) / 
      CASE 
        WHEN DATEDIFF(year, created_date, '2022-09-01') < 1 THEN 1.0
        ELSE DATEDIFF(year, created_date, '2022-09-01') 
      END AS Frequency,
    SUM(ct.GMV) AS Monetary
  FROM Customer_Registered cr
  JOIN Customer_Transaction ct ON cr.id = ct.CustomerID
  WHERE created_date IS NOT NULL AND ct.GMV > 0
  GROUP BY cr.id, cr.created_date
),

-- STEP 2
-- IQR THRESHOLDS
-- Compute Quartiles for each R, F, M metric using PERCENTILE_DISC
IQR AS (
  SELECT id, Contract_age, Recency, Frequency, Monetary,
    PERCENTILE_DISC(0.25) WITHIN GROUP (ORDER BY Recency) OVER() AS R_Q1,
    PERCENTILE_DISC(0.50) WITHIN GROUP (ORDER BY Recency) OVER() AS R_Q2,
    PERCENTILE_DISC(0.75) WITHIN GROUP (ORDER BY Recency) OVER() AS R_Q3,
    MAX(Recency) OVER() AS R_Q4,
    
    PERCENTILE_DISC(0.25) WITHIN GROUP (ORDER BY Frequency) OVER() AS F_Q1,
    PERCENTILE_DISC(0.50) WITHIN GROUP (ORDER BY Frequency) OVER() AS F_Q2,
    PERCENTILE_DISC(0.75) WITHIN GROUP (ORDER BY Frequency) OVER() AS F_Q3,
    MAX(Frequency) OVER() AS F_Q4,
    
    PERCENTILE_DISC(0.25) WITHIN GROUP (ORDER BY Monetary) OVER() AS M_Q1,
    PERCENTILE_DISC(0.50) WITHIN GROUP (ORDER BY Monetary) OVER() AS M_Q2,
    PERCENTILE_DISC(0.75) WITHIN GROUP (ORDER BY Monetary) OVER() AS M_Q3,
    MAX(Monetary) OVER() AS M_Q4
  FROM RFM
),

-- STEP 3
-- RFM SCORING
-- R: Inverted. fewer days since purchase = score 4
-- F: ascending. higher frequency = score 4
-- M: ascending. higher monetary = score 4
RFM_RANK AS (
  SELECT id,
    CASE 
      WHEN Recency <= R_Q1 THEN 4
      WHEN Recency <= R_Q2 THEN 3
      WHEN Recency <= R_Q3 AND R_Q3 < R_Q4 THEN 2
      ELSE 1 
    END AS R_Score,
    CASE 
      WHEN Frequency <= F_Q1 THEN 1
      WHEN Frequency <= F_Q2 THEN 2
      WHEN Frequency <= F_Q3 AND F_Q3 < F_Q4 THEN 3
      ELSE 4 
    END AS F_Score,
    CASE 
      WHEN Monetary <= M_Q1 THEN 1
      WHEN Monetary <= M_Q2 THEN 2
      WHEN Monetary <= M_Q3 AND M_Q3 < M_Q4 THEN 3
      ELSE 4 
    END AS M_Score
  FROM IQR
),

-- STEP 4
-- COMBINE SCORES
FINAL_OUTPUT AS (
  SELECT 
    i.id, i.Contract_age, i.Recency, i.Frequency, i.Monetary,
    CONCAT(r.R_Score, r.F_Score, r.M_Score) AS RFM_Score
  FROM IQR i
  JOIN RFM_RANK r ON i.id = r.id
)
SELECT * FROM FINAL_OUTPUT;
